###############
Ignorar fitxers
###############

Sovint hi ha fitxers (i directoris) que no voldrem que Git gestioni.
Per exemple, els fitxers temporals que ens generen alguns editors
(normalment acabats en ``~``), o fitxers amb codi objecte o
executables (ex. fitxers objecte ``.o`` de C o els compilats de java
``.class``)

Podem indicar a Git que ignori aquests fitxers creant/modificant el
fitxer ``project/.gitignore``. Per exemple, el següent fitxer
indicaria a Git que ignori els fitxers temporals d'edició, els
resultats de compilació, els comprimits, i el contingut del directori
on es construeix l'aplicació i el temporal.

.. code-block::

    $ cat projecte/.gitignore
    ## Fitxers a ignorar pel Git

    # Fitxers d'edició
    *~

    # Fitxers intermitjos
    *.[oa]
    *.class
    *.pyc

    # paquets i comprimits
    *.7z
    *.dmg
    *.gz
    *.iso
    *.jar
    *.rar
    *.tar
    *.tar.gz
    *.zip

    # contingut de directoris complets
    build/
    tmp/

Atenció: en cas que ja estiguis registrant fitxers que has marcat per
ignorar al ``.gitignore``, hauràs d'eliminar-los manualment. Per
exemple, si ja havies fet *commit* d'un ``.class`` pots eliminar-lo
amb

.. code-block:: console

    projecte$ git rm --cached *.class

Ves amb compte perquè aquesta comanda és una de les poques que
eliminen informació permanentment del repositori de Git!

Trobaràs informació més completa de com ignorar fitxers a `aquest
article <https://help.github.com/articles/ignoring-files>`_
