I en cas que no, que passa sí…¶

«Si tens just 18 anys llavors tot just arribes a la majoria d’edat; si tens més de 18 anys llavors ets major d’edat; altrament ets menor d’edat»

De vegades ens interessa expressar més d’una condició en una estructura condicional.

La frase anterior, la podríem dibuixar de la següent manera:

En Java, aquest tros de codi el podem codificar com:

if (edat == 18) {
    System.out.println("Tot just major d'edat");
} else {
    if (edat > 18) {
        System.out.println("Ets major d'edat");
    } else {
        System.out.println("Ets menor d'edat");
    }
}

Fixa’t que tenim un if dins d’un else! És clar, les estructures condicionals són també instruccions i, com a tals, les podem posar allà on tingui sentit una instrucció. Per exemple, dins d’un bloc d’una altra condició.

En programació, quan una estructura es troba dins d’una altra del mateix tipus és sol dir que les estructures es troben niades. És una mala traducció del terme anglès nested però no tenim al nostre idioma cap paraula que expressi amb precisió aquest concepte. En alguns llocs trobaràs que fan servir imbricades. Bé, el significat és molt més proper que el dels nius però tampoc no és precís.

Aquesta situació de tenir més d’una condició, però, és tant comuna que molts llenguatges ofereixen una manera més compacta d’expressar-les. En el cas de Java, ho podem fer de la següent manera:

if (edat == 18) {
    System.out.println("Tot just major d'edat");
} else if (edat > 18) {
    System.out.println("Ets major d'edat");
} else {
    System.out.println("Ets menor d'edat");
}

Aquesta nova versió estalvia dues línies de codi i un nivell d’indentació. Amb tot, només és una millora estètica. Els dos programes funcionen exactament iguals. Amb tot, com passa amb la indentació, a mida que anem programant veurem que tota millora en la legibilitat del codi serà molt valuosa.

Per cert, podem indicar tants else if com ens calguin i no és obligatori acabar amb un else.
